cd ~/.ssh
Comprobar si existe una clave SSH en el equipo

Si existe una clave SSH en el equipo nos posicionará en el directorio .ssh y sino nos mostrará un mensaje diciendo que no existe tal directorio.

ls
Listar carpetas y archivos contenidos en un determinado directorio

---

mkdir key_backup
Crear directorio "key_backup"

Con este comando creamos un directorio llamado "key_backup" para guardar una copia de las claves que ya existen para posteriormente borrarlas y crear una nueva clave SSH para Git.

cp id_rsa* key_backup
Copiar archivos en la carpeta "key_backup"

Copiamos los archivos que empiecen con "id_rsa" dentro de la carpeta "key_backup". Esto copiará los archivos "id_rsa" y "id_rsa.pub" a la carpeta "key_backup".

rm id_rsa*
Borrar archivos

Esto borrará los archivos que empiecen con "id_rsa" del directorio actual.

ssh-keygen -t rsa -C "edujjalvarez@gmail.com"
Crear nueva clave SSH con un email válido

Este proceso solicitará ingresar una ruta para almacenar la clave generada y una contraseña dos veces para poder confirmarla.

mkdir git-repositories

mkdir primer-proyecto

Posicionarse en la carpeta "primer-proyecto"

---

git init
Iniciar el repositorio

---

touch README.md
Agregar archivo "README.md" al repositorio

---

git add README.md
Agregar el archivo "README.md" a la lista de cambios a subir

Esto coloca el archivo "README.md" en el estado STAGED. Que es el estado intermedio de los tres estados que maneja Git (MODIFIED, STAGED, COMMITED).

git commit -m "first commit"
Realizamos un comit para pasar los cambios del estado STAGED a COMMITED

En este punto los cambios ya quedan guardados en el repositorio.

git remote add origin https://github.com/edujjalvarez/primer-proyecto.git
Agregamos los cambios al repositorio remoto "primer-proyecto" dentro de GitHub

En este punto los cambios ya quedan guardados en el repositorio.

git push -u origin master
Empujamos los cambios en el repositorio remoto "primer-proyecto"

---

git init
Coniverte cualquier carpeta o directorio de nuestro sistema de archivos, que puede estar almacenada en cualquier lugar del equipo, en un repositiorio de Git.

---

git add *.cs
Agrega todos los archivos con extensión ".cs" contenidos en un directorio convertido en repositorio de Git a la sección STAGED donde quedan pendientes para el guardarse en el próximo COMMIT.

---

git add -A
Agrega todos los archivos y carpetas de un repositorio a la sección STAGED.

Todos se refiere a los archivos y carpetas NUEVOS, MODIFICADOS y ELIMINADOS. Es equivalente a "git add --all".

git add .
Agrega todos los archivos y carpetas de un repositorio a la sección STAGED.

Todos se refiere a los archivos y carpetas NUEVOS, MODIFICADOS y ELIMINADOS.

git add --ignore-remmoval .
Agrega todos los archivos y carpetas NUEVOS y MODIFICADOS (NO ELIMINADOS) de un repositorio a la sección STAGED.

---

git add -u
Agrega todos los archivos y carpetas MODIFICADOS y ELIMINADOS (NO NUEVOS) de un repositorio a la sección STAGED.

Es equivalente a "git add --update".

git clone https://github.com/twbs/bootstrap.git
Clonar un repositorio en nuestro sistema de archivos.

Esto crea una directorio nuevo para el repositorio en la ubicación donde estemos posicionados. En este caso estoy clonando el repositorio oficial de Bootstrap.

git clone https://github.com/twbs/bootstrap.git nombre-carpeta
Clonar un repositorio en nuestro sistema de archivos.

A diferencia del comando anterior esto clonara el repositorio dentro de la carpeta "nombre-carpeta", creandola si fuese necesario.

Ciclo de vida de un archivo Git
Estados de un archivo:

Tracked: Rastreado.

Untracked: No rastreado.

Unmodified: No modificado.

Modified: Modificado.

Staged: Staged.

Commited: En este punto al hacer un COMMIT el archivo vuelve al estado MODIFIED.

git status
Checkear en que estado del ciclo de vida se encuentran los archivos.

---

git reset HEAD README.md
Quitar el archivo del STAGE para que no se envie en el próximo COMMIT.

---

git reset HEAD borrar/
Quitar el directorio "borrar" y todo su contenido del STAGE para que no se envie en el próximo COMMIT.

El archivo .gitignore

Este archivo se utiliza para que el sistema ignore ciertos archivos al realizar cambios. Esto se da frecuentemente cuando se utilizan IDE's para el desarrollo ya que estos generan archivos temporales que utiliza cada equipo para trabajar correctamente pero no forman parte del código fuente puro del proyecto. Existen muchos patrones para ignorar archivos y directorios que se deben colocar en un archivo sin nombre y con extensión ".gitignore".

La primera línea es obligatorio en todo archivo ".gitignore".

git diff
Muestra los cambios que se hicieron en los archivos.

Nota: "git diff" solo muestra las diferencias de los archivos modificados que NO ESTAN EN EL STAGE. Esto quiere decir que si hubiera un archivo que ha sido modificado y puesto en el STAGE dichos cambios (diferencia) no se mostrarn con el comando "git diff".

git diff --staged
Muestra los cambios que se hicieron en los archivos.

Nota: A diferencia del comando anterior este si muestra los cambios en los archivos que hayan sido modificados y puestos en el STAGED.

git commit -v -m "Esto es una descripción del COMMIT"
Envia un COMMIT con el detalle de los cambios (diferencias) entre archivos.

Esto agrega un detalle adicional adjunto al COMMIT donde se detalla los cambios que se hicieron en los archivos. El detalle es el mismo que nos mostraría el comando "git diff" o "git diff --sataged", aparenciendo con un signo "+" adelante de la línea si es una línea que se agrego y con un "-" si es una línea que se quito.

git commit -m "Esto es una descripción del COMMIT" -a
Envia un COMMIT saltandose el STAGE.

Esto envia los cambios de los archivos modificados aunque estos no hayan sido puestos en el STAGE con el comando "git add".

git rm archivo-borrar.txt
Borrar archivo.

Esto borra el archivo del directorio de trabajo. Aunque el archivo haya sido borrado de manera física del directorio de trabajo GIT sigue rastreandolo. Con este comando se borra definitivamente y ya no queda rasterado por GIT. Si no fue borrado de manera física este comando lo borra físicamente y lo coloca en el STAGE para el próximo COMMIT. Si el archivo ya esta en el STAGE este comando no funciona.

git rm -f archivo-borrar.txt
Borrar archivo.

Esto borra el archivo aunque este en el STAGE. Se fuerza el borrado.

git rm --cached archivo-borrar.txt
Borrar archivo.

Esto borra el archivo del STAGE pero no físicamente (del disco rígido).

git rm \*~
Borrar archivo.

Esto borra todos los archivos que finalizen con "~".

git rm log/ \*.log
Borrar archivo.

Esto borra todos los archivos con extensión ".log" que estan dentro de la carpeta "log".

git mv nombre-viejo.txt nombre-nuevo.txt
Renombrar archivo.

Esto renombra el archivo.
